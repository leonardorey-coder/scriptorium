Basado en **UNIDAD 1:**
- **Definición de Pruebas de Software**
  - Concepto formal
  - Procesos involucrados
  - Verificación vs. Validación
  
- **¿Qué es una Prueba de Software?**
  - Son procesos donde se evalúa y verifica que un producto o programa de software cumpla con los requisitos especificados
  - El software funciona como se espera
  
- **Importancia de las Pruebas**
  - Identificar y corregir errores antes del lanzamiento

### **6. GLOSARIO TÉCNICO** (2-3 páginas)

**Mínimo 40 palabras técnicas** (el profesor solicitó 10, pero incluir más):

- **Caso de Prueba:** Conjunto de pasos diseñados para verificar una funcionalidad
- **Defecto/Bug:** Diferencia entre comportamiento esperado y actual
- **Cobertura de Pruebas:** Porcentaje de código ejecutado durante las pruebas
- **Regression Testing:** Pruebas para verificar que cambios no rompieron funcionalidad existente
- **Test Plan:** Documento que describe la estrategia de pruebas
- **Validación:** Verificar que el producto cumple requisitos del usuario
- **Verificación:** Verificar que el producto cumple especificaciones técnicas
- **QA (Quality Assurance):** Aseguramiento de la calidad
- **Black Box Testing:** Pruebas sin conocimiento del código interno
- **White Box Testing:** Pruebas con conocimiento del código interno
- **Smoke Testing:** Pruebas rápidas para verificar funcionalidad básica
- **Stress Testing:** Pruebas bajo condiciones extremas
- **Performance Testing:** Evaluación de rendimiento y velocidad
- **Security Testing:** Pruebas de vulnerabilidades
- **Usability Testing:** Evaluación de facilidad de uso
- **Acceptance Testing:** Pruebas finales antes de lanzamiento
- **Unit Testing:** Pruebas de componentes individuales
- **Integration Testing:** Pruebas de integración entre componentes
- **Boundary Testing:** Pruebas en límites de valores válidos
- **End-to-End Testing:** Pruebas del flujo completo del sistema

(Continúa con más términos)
